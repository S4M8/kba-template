Password Reset Process for Active Directory Users

Step-by-step guide for help desk technicians to reset user passwords.

Process ID: PROC-AD-001
Application: Active Directory
Last Updated: 2025-07-16
Author: AI Assistant

Overview

This document outlines the standard operating procedure for help desk technicians to reset user passwords within Active Directory. This process ensures security compliance and efficient resolution of user access issues.

Process

Before You Begin
Ensure you have appropriate administrative privileges in Active Directory Users and Computers (ADUC) and have verified the user's identity according to company policy.

Process Steps

  1. Open Active Directory Users and Computers (ADUC):
    On your administrative workstation, open Server Manager, then navigate to Tools > Active Directory Users and Computers.
  2. Locate the User Account:
    In the ADUC console, browse to the Organizational Unit (OU) where the user's account is located. You can also use the "Find" feature (Ctrl+F) to search for the user by name.
  3. Reset Password:
    Right-click on the user account, select "Reset Password...".
  4. Enter New Password:
    In the "Reset Password" dialog box, enter a new, strong password in both the "New password" and "Confirm password" fields. Ensure the password meets complexity requirements.
  5. Set User Options:
    Typically, you should check "User must change password at next logon" to enforce a new password creation by the user. Uncheck "Account is locked out" if it is checked.
  6. Confirm Reset:
    Click "OK" to complete the password reset. Inform the user of their temporary password and instruct them to change it immediately upon logging in.
Important Note
Never share temporary passwords via unencrypted channels (e.g., unencrypted email or chat). Always communicate them securely, preferably verbally or through a secure, one-time password system.

PowerShell Command for Password Reset

For experienced technicians, a password can also be reset using PowerShell:

Set-ADUser -Identity "username" -NewPassword (Read-Host -AsSecureString "Enter new password") -ChangePasswordAtLogon $true

Replace "username" with the actual user's SAMAccountName.

Verification

Success Indicators

The password reset is successful when:

  • The user is able to log in with the new temporary password.
  • The user is prompted to change their password upon first login (if "User must change password at next logon" was checked).
  • The user confirms successful access to their resources.
